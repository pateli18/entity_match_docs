### 1. Install the Application in Your Workspace
<Info>This only needs to be done once per workspace</Info>

Search for "EntityMatcher" in the [Google Workspace Marketplace](https://workspace.google.com/marketplace/search/EntityMatcher) and `Install` the application.

<Frame caption="Install EntityMatcher in Google Sheets">
  <img src="/screenshots/gsheets/plugin_install.png" alt="Install EntityMatcher in Google Sheets" />
</Frame>

### 2. Enable the Plugin in Your Worksheet
<Info>This only needs to be done once per worksheet</Info>

Click `Extensions > Add-ons > Manage add-ons` in the menu bar.

<Frame caption="Manage Add-Ons">
  <img src="/screenshots/gsheets/extensions_manage_add_ons.png" alt="Manage Add-Ons" />
</Frame>

Then click the three dots in the top right corner of the EntityMatcher add-on and select `Use in this document`.

<Frame caption="Use in this document">
  <img src="/screenshots/gsheets/plugin_enabled.png" alt="Use in this document" />
</Frame>

<Warning>You may need to exit and re-open the worksheet after enabling it to have the changes take effect</Warning>

### 3. Get and Set Your API Key
<Info>This only needs to be done once per workspace</Info>

<Snippet file="api-key.mdx" />

Once you've retrieved the API key, add it to the plugin by clicking `Extensions > EntityMatcher > API Key` in the menu bar.

<Frame caption="Open API Key Dialog">
  <img src="/screenshots/gsheets/api_key_menu.png" alt="Open API Key Dialog" />
</Frame>

Paste in your API Key and click `Ok`.

<Frame caption="Add API Key">
  <img src="/screenshots/gsheets/api_key_dialog.png" alt="Add API Key" />
</Frame>

### 4. Start a Match Job

If all the data that you want is in the same worksheet, use the `=ENTITYMATCH` custom formula. Here's an example formula:

```
=ENTITYMATCH(A1:E51,"id", google!A1:E44, "id", "test_google_sheets_matching", True)
```

- The first parameter covers the `base_dataset` which is the dataset with every record that you want to be matched
- The second parameter is the `base_id_column` which is the column in the `base_dataset` that contains the unique identifier for each record
- The third parameter covers the `match_dataset` which is the dataset with the records that you want to match against the `base_dataset`. This can be the same as the `base_dataset` if you are, for example, trying to de-duplicate the dataset
- The fourth parameter is the `match_id_column` which is the column in the `match_dataset` that contains the unique identifier for each record
- The fifth parameter is the `match_job_name` which is the name of the match job that you want to use. This should be unique within the work sheet if you are using multiple match jobs
- The sixth parameter is an **optional** parameter that is by default `False`. Set it to `True` to receive an email when the match job is complete

<Warning>For the `base_dataset` and `match_dataset`, select the entire datasets and not the individual rows</Warning>

Once you press enter, the relevant cells will be populated with `Loading...`

### 4. Refresh Values

You can't just press enter to refresh the values because of the way Google caches results. Instead, click `Extensions > EntityMatcher > Match Jobs` in the menu bar.

<Frame caption="Open Match Jobs Dialog">
  <img src="/screenshots/gsheets/match_menu.png" alt="Open Match Jobs Dialog" />
</Frame>

Find your match job (it should have match the `match_job_name` that you specified in the formula) and click `Refresh`.

<Frame caption="Refresh Match Job">
  <img src="/screenshots/gsheets/match_refresh_job.png" alt="Refresh Match Job" />
</Frame>
